Gretzky Example Components

This is a page consisting of various pre-built gretzky components using simple markup/styling to demonstrate just how the widgets can work together.

Viewing the widget sources and various code snippets below will show you how primitive modules can be built up into larger functioning widgets.

Combined functionality stems from the use of events between parent/children widgets. Take note of the various events each component actively emits or listens for by default - a clean event API is key to an easily-maintainable widget.

Switch

Description

Simple component that holds an on/off state.

Options
  • active (boolean)
    Switch active on init (default: false)
  • class (string)
    Class to apply while active (default: 'is-active')
  • on (string)
    String to replace inside switch when active (default: null)
  • off (string)
    String to replace inside switch when disabled (default: null)
Working Example
Switch
Code
<div data-widget="switch" data-active="true" data-on="Switch (Enabled)" data-off="Switch (Disabled)">Switch</div>
Events
Emits
  • switch:was-changed
    Switch state has been updated (but not necessarily changed)
  • switch:is-enabled
    Switch state has been set to 'enabled'
  • switch:is-disabled
    Switch state has been set to 'disabled'
Listens For
  • switch:enable()
    Sets the switch state to 'enabled'
  • switch:disable()
    Sets the switch state to 'disabled'

Toggle

Description

Stores and manages the state of two or more Switch components.

Options
  • maxActive (number)
    Max switches active at one time (default: 1)
  • allowEmpty (boolean)
    Allow state in which all switches can be disabled (default: false)
  • hasDefault (boolean)
    Denotes if an option should be the default option (e.g. for 'all' filters) (default: null)

    Accordion

    Description

    Hides/displays a section of content based on an 'active' Switch state.

    Options
    • active (boolean)

    • animIn (string)
      Animation name to play when accordion opens (default: 'fadeInUp')
    • animOut (string)
      Animation name to play when accordion closes (default: 'fadeOutDown')

            Input

            Description

            Wrapper for the standard text input fields (text, password, etc). Provides validation with AOT configurations.

            Options
            • type (string)
              Validation set to run on field (default: 'all')
            • valid (string)
              Class applied when field is valid (default: 'content')
            • invalid (string)
              Class applied when field is invalid (default: 'is-invalid')
            Working Example (Check the console)
            Code
            <input type="text" name="username" placeholder="Username" data-widget="input" data-type="length">
            Events
            Emits
            • input:validChange
              The 'valid' state of the field has changed.
            • input:valid
              The field value has been successfully validated.
            • input:invalid
              The field value has failed validation.
            Listens For

              Digits

              Description

              Wrapper for displayed numerical values on the site. Simply animates from 0 to the target number. On hover, the animation will spin up again (if not already animating).

              Options
              • speed (number)
                Speed of animation, ranged 2 - 100 (default: 25)

                Sifter

                Description

                Hides/displays content based on various filter inputs.

                Two styles of filters are available: additive and non-additive. Non-additive is default and will return all items for the selected filters. Additive combines filters and returns items matching EXACTLY those filter specifications. For example, select 'luxury' and 'kids' in the various examples to see the difference.

                Options
                • additive (boolean)
                  When true, only items matching the filters exactly are returned (default: false)
